[과제] 개발 워크스테이션 구축 및 Docker/Git 실습 보고서
1. 프로젝트 개요
미션 목표: 리눅스 터미널 조작, 권한 관리, Docker를 이용한 컨테이너 기반 웹 서버 구축 및 Git/GitHub 연동을 통한 버전 관리 역량 습득.

2. 실행 환경
OS: (예: macOS Sequoia / Ubuntu 22.04 등 본인 환경 작성)

Shell/Terminal: (예: Zsh / iTerm2)

Docker 버전: docker --version 결과값 입력

Git 버전: git --version 결과값 입력

3. 수행 항목 체크리스트
[ ] 터미널 기본 조작 로그 기록

[ ] 파일/디렉토리 권한 변경 실습 및 증거

[ ] Docker 설치 및 데몬 동작 확인

[ ] Docker 기본 운영 명령(이미지, 컨테이너, 로그, 리소스) 수행

[ ] hello-world 및 ubuntu 컨테이너 실행 실습

[ ] 커스텀 Dockerfile 기반 웹 서버 컨테이너 제작

[ ] 포트 매핑 및 접속 검증

[ ] 바인드 마운트 및 Docker 볼륨 영속성 검증

[ ] Git 설정 및 GitHub 저장소 연동

4. 상세 수행 내용 및 검증 로그
4.1 터미널 조작 및 권한 실습
수행 명령: pwd, ls -al, cd, mkdir, cp, mv, rm, cat, touch

권한 변경 실습: chmod를 이용한 755, 644 등 권한 변경 전/후 비교 (ls -l)

Bash
# 예시: 파일 생성 및 권한 변경
touch test.txt
ls -l test.txt
chmod 755 test.txt
ls -l test.txt
4.2 Docker 기본 운영
설치 확인: docker --version, docker info

이미지 및 컨테이너: docker images, docker ps -a

컨테이너 실행: hello-world 실행 결과 및 ubuntu 컨테이너 내부 진입 실습 (ls, echo)

4.3 커스텀 웹 서버 제작 (Dockerfile)
선택 베이스: (예: NGINX / Ubuntu 등 선택한 베이스 명시)

커스텀 포인트: (예: index.html 수정, 환경 변수 추가 등)

빌드/실행 명령:

Bash
docker build -t my-custom-web .
docker run -d -p 8080:80 my-custom-web
접속 검증: 브라우저 접속 스크린샷 또는 curl -I localhost:8080 결과

4.4 데이터 영속성 (마운트/볼륨)
바인드 마운트: 호스트 파일 수정 시 컨테이너 반영 여부 확인

Docker 볼륨: 볼륨 생성, 컨테이너 연결, 컨테이너 삭제 후 데이터 유지 증명 명령 기록

4.5 Git & GitHub 설정
Git 설정 확인: git config --list (사용자 이름/이메일 확인)

연동 증거: GitHub Repository 연결 상태 및 Push 완료 내역
