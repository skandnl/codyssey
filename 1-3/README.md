# Mini NPU Simulator

MAC(Multiply-Accumulate) 연산 기반 패턴 판별 Python 콘솔 애플리케이션.  
십자가(Cross)와 X 패턴을 필터와 비교하여 유사도를 계산하고 판별하는 Mini NPU 시뮬레이터.

---

## 실행 방법

### 요구 사항

- Python 3.8 이상
- 외부 라이브러리 없음 (표준 라이브러리만 사용)

python main.py

## 구현 요약

### 1. 데이터 구조 — Pattern 클래스

```python
class Pattern:
    def __init__(self, n: int):
        self.n = n
        self.data = [[0.0] * n for _ in range(n)]
    def set_value(self, row, col, value): ...
    def get_value(self, row, col) -> float: ...
```

N×N 크기의 2차원 배열을 저장한다.  
`from_2d_list()` 클래스 메서드로 JSON 배열에서 직접 생성 가능.

### 2. MAC 연산 — 반복문 직접 구현

```python
def mac_compute(pattern, filter_p) -> float:
    score = 0.0
    for i in range(n):
        for j in range(n):
            score += pattern.get_value(i, j) * filter_p.get_value(i, j)
    return score
```

NumPy 등 벡터화 라이브러리 없이 **이중 반복문**으로 직접 구현.  
입력 패턴과 필터를 같은 위치끼리 곱하고 모두 더해 유사도 점수를 반환.

### 3. 라벨 정규화 (표준화)

```python
def normalize_label(label: str) -> str:
    normalized = label.strip().lower()
    if normalized in ("+", "cross"):  return "Cross"
    elif normalized in ("x",):        return "X"
    return label
```

| 입력 값 | 표준 라벨 |
|---------|----------|
| `+`     | `Cross`  |
| `cross` | `Cross`  |
| `x`     | `X`      |
| `X`     | `X`      |

`data.json`의 `expected` 필드와 `filters` 키를 모두 이 함수로 정규화한 후 비교.  
정규화를 적용하면 `"+" == "Cross"` 같은 오판 없이 일관된 비교가 가능하다.

### 4. 동점 처리 정책 — epsilon 기반

```python
EPSILON = 1e-9

def judge(score_a, score_b, ...) -> str:
    if abs(score_a - score_b) < EPSILON:
        return "UNDECIDED"
    ...
```

부동소수점 연산에서는 수학적으로 동일한 값도 이진 표현 차이로 아주 작은 오차가 발생할 수 있다.  
예를 들어 `0.1 + 0.2 ≠ 0.3`이 Python에서 발생하는 것처럼, MAC 누적 연산도 순서에 따라 오차가 누적된다.  
`abs(score_a - score_b) < 1e-9` 조건으로 실질적 동점을 안전하게 UNDECIDED로 처리한다.

### 5. 보너스 — 패턴 생성기

```python
def generate_cross_pattern(n): ...
def generate_x_pattern(n): ...
```

크기 N을 입력하면 N×N 십자가/X 패턴을 자동 생성.  
성능 분석 측정에 이 함수로 생성한 패턴을 재활용한다.

## 결과 리포트

### FAIL 케이스 원인 분석

이번 실행에서 **FAIL 0건**이 달성되었다. 그 이유는 다음 세 가지 정책이 복합적으로 작동했기 때문이다.

**1. 라벨 정규화 정책**  
`data.json`의 `expected` 필드는 `"+"`, `"x"` 형태로 저장되어 있고, `filters` 키는 `"cross"`, `"x"` 형태다.  
이 두 표현을 `normalize_label()` 함수를 통해 모두 `"Cross"`, `"X"` 표준 라벨로 변환한 뒤 비교하므로, 라벨 불일치로 인한 FAIL이 발생하지 않는다.  
만약 정규화 없이 `"+" == "Cross"`를 직접 비교하면 전체 케이스가 FAIL 처리된다.

**2. epsilon 기반 동점 처리 정책**  
정수 배열로 구성된 현재 데이터에서는 MAC 연산 결과가 명확히 정수값이므로 부동소수점 오차가 거의 발생하지 않는다.  
그러나 실수 가중치가 포함된 실제 AI 필터에서는 `score_a - score_b`가 1e-15 수준의 차이를 보일 수 있다.  
`epsilon = 1e-9` 기준으로 이를 UNDECIDED로 처리하면, 의미 없는 부동소수점 차이로 인한 오판을 방지한다.

**3. 스키마 검증 정책**  
필터 누락, 크기 불일치, 키 형식 오류 발생 시 프로그램을 중단하지 않고 해당 케이스만 FAIL 처리 후 계속 진행한다.  
현재 데이터는 모두 정상 스키마를 만족하므로 이 케이스에 해당하는 FAIL도 발생하지 않았다.

**결론:** FAIL 0건은 데이터의 완결성 + 라벨 정규화 + epsilon 정책이 함께 작동한 결과다.  
만약 어느 하나라도 적용하지 않는다면 FAIL이 다수 발생할 수 있다.

### 성능 표 해석 + O(N²) 시간 복잡도 분석

#### 실행 결과 예시

| 크기  | 평균 시간(ms) | 연산 횟수(N²) |
|-------|-------------|-------------|
| 3×3   | 0.0012      | 9           |
| 5×5   | 0.0029      | 25          |
| 13×13 | 0.0174      | 169         |
| 25×25 | 0.0611      | 625         |

#### O(N²) 근거

MAC 연산은 N×N 배열을 두 개의 중첩 반복문으로 순회한다.

```
for i in range(N):       # N번 반복
    for j in range(N):   # N번 반복
        score += ...     # 1번 연산
```

따라서 총 연산 횟수 = N × N = **N²**회.

N이 3 → 25로 약 8.3배 증가할 때 연산 횟수는 9 → 625로 약 **69배** 증가한다.  
측정 시간도 0.0012 → 0.0611 ms로 약 **51배** 증가하여 O(N²) 예측과 근사하게 일치한다.  
(실측 배율이 이론보다 낮은 이유는 CPU 캐시 최적화, 반복문 오버헤드 비율 변화 등의 영향이다.)

실제 AI 모델에서 256×256 필터를 사용하면 연산 횟수는 65,536회이고, 이를 수천 개의 필터로 수행하면 수억 번의 MAC이 필요하다. 이것이 NPU (병렬 처리 전용 칩)가 필요한 이유다.
